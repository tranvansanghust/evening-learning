### Use in FastAPI
```python
from fastapi import Depends
from app.database import get_db
from app.models import User

@app.get("/users/{user_id}")
async def get_user(user_id: int, db: Session = Depends(get_db)):
    return db.query(User).get(user_id)
```

## The 8 Models

### 1. User
Telegram user accounts with learning progress tracking.

**Table:** `users`
**Columns:** user_id, telegram_id, username, level, created_at, updated_at
**Key Features:**
- Unique telegram_id for bot integration
- Level system (0-3: beginner to expert)
- UTC timestamps with automatic updates

### 2. Course
Educational courses from various sources.

**Table:** `courses`
**Columns:** course_id, name, description, source, source_id, total_lessons, created_at
**Key Features:**
- Support for Udemy and internal courses
- External source_id for API integration
- Composite index on (source, source_id)

### 3. UserCourse
Many-to-many enrollment relationship with progress tracking.

**Table:** `user_courses`
**Columns:** user_course_id, user_id, course_id, status, started_at, completed_at
**Key Features:**
- Status tracking (PASS, FAIL, IN_PROGRESS)
- Course completion timestamps
- Cascade delete for data integrity

### 4. Lesson
Individual lessons within a course.

**Table:** `lessons`
**Columns:** lesson_id, course_id, sequence_number, title, description, content_url, estimated_duration_minutes
**Key Features:**
- Ordered lessons via sequence_number
- Content URL for frontend rendering
- Duration estimation for progress tracking

### 5. Concept
Learning objectives/concepts within lessons.

**Table:** `concepts`
**Columns:** concept_id, lesson_id, name, description
**Key Features:**
- Link to quiz answers for mastery tracking
- Basis for quiz question generation

### 6. QuizSession
Quiz sessions with conversation history.

**Table:** `quiz_sessions`
**Columns:** session_id, user_id, lesson_id, status, messages, started_at, completed_at
**Key Features:**
- JSON storage for conversation history
- Status tracking (active, completed)
- One-to-one relationship with summary

### 7. QuizAnswer
Individual Q&A pairs within a quiz session.

**Table:** `quiz_answers`
**Columns:** answer_id, session_id, concept_id, question, user_answer, is_correct, engagement_level, created_at
**Key Features:**
- LLM evaluation (correct/incorrect)
- Engagement tracking (low, medium, high)
- Links question to concept being tested

### 8. QuizSummary
Post-quiz summaries generated by LLM.

**Table:** `quiz_summaries`
**Columns:** summary_id, session_id, user_course_id, concepts_mastered, concepts_weak, created_at
**Key Features:**
- JSON storage for mastered concepts
- JSON storage for weak concepts with explanations
- One-to-one with quiz session

## Usage Examples

### Create a User
```python
from app.models import User
from app.database import SessionLocal

db = SessionLocal()
user = User(telegram_id="123456789", username="john", level=0)
db.add(user)
db.commit()
```

### Enroll User in Course
```python
from app.models import UserCourse

enrollment = UserCourse(
    user_id=user.user_id,
    course_id=course.course_id,
    status="IN_PROGRESS"
)
db.add(enrollment)
db.commit()
```

### Create Quiz Session
```python
from app.models import QuizSession

session = QuizSession(
    user_id=user.user_id,
    lesson_id=lesson.lesson_id,
    status="active",
    messages=[
        {"role": "bot", "content": "What is a variable?"},
        {"role": "user", "content": "A container for data"}
    ]
)
db.add(session)
db.commit()
```

### Get User with All Enrollments
```python
from sqlalchemy.orm import joinedload

user = db.query(User).options(
    joinedload(User.user_courses).joinedload(UserCourse.course)
).filter(User.user_id == user_id).first()
```

## Troubleshooting

### Common Issues

**"No module named 'sqlalchemy'"**
```bash
pip install sqlalchemy==2.0.23
```

**"Foreign key constraint error"**
- Ensure parent records exist before adding children
- Check cascade delete rules if deleting fails

**"N+1 query problem"**
- Use `joinedload()` to eager load relationships
- Check query performance with `echo=True`

**"Stale session error"**
- Always use `with db:` or `try/finally` to ensure close
- Or use FastAPI dependency injection
